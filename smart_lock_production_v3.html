━━━ All event_type values ━━━

  "door_opened"      — relay trigger, door khuli
  "door_closed"      — reed switch: door physically band hui
  "door_ajar"        — open_alert_after_sec baad bhi open
  "failed_attempt"   — wrong pin / unknown fingerprint / unknown rfid
  "lockout_started"  — brute force: too many failed attempts
  "lockout_ended"    — lockout duration complete
  "doorbell"         — physical button press + camera snap
  "tamper"           — vibration/force detected on lock
  "battery_low"      — battery percent threshold crossed
  "battery_critical" — critical level (e.g. 10%)
  "device_online"    — ESP32 reconnected to MQTT
  "device_offline"   — LWT (Last Will Testament) triggered
  "ota_started"      — firmware update started
  "ota_completed"    — firmware update done
  "user_added"       — new shared user added
  "user_removed"     — shared user removed
  "pin_created"      — new pin added
  "pin_revoked"      — pin manually revoked
  "settings_changed" — lock settings updated
GSI-1 (user_id#timestamp) se Flutter app mein "Meri history" page fast load hogi. GSI-2 (event_type#timestamp) se "Sirf failed attempts" ya "Sirf doorbell events" filter instantly milega.
